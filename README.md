# 7-2-Portfolio
Client and Software Design:
The Gaming Room was looking for a web game called "Draw It or Lose It". This was modeled on the game on the TV show "Win, Lose, or Draw". The game was required to work on many different devices and allow several teams to compete to guess drawings in a pre-made pool.

Documentation Development: 
I have done an excellent job in documenting the design constraints, requirements, and client requirements in my documentation. The system architecture and the domain model was well documented.

Design Document Process:
The process of working through the design document helped me with the process of coding.

Revisions and Improvements: 
If I had to make a change in the documentation I would improve on the evaluation section by providing a more detailed comparison of operating platforms.

User Needs Interpretation: 
This helped to correctly translate the requirements such as ensuring all names were unique and that there would be only one instance of the class, both of which were required for the correct functioning and good performance of the application. To understand the user's needs and wants is important, as it can ensure that the software is easy to use, works well, and does what it is meant to do, in turn leading to higher user satisfaction.

Software Design Approach: 
The approach used for this software design was a modular design that utilized a microservice-based architecture. For future designs I will be employing structured design documents and incorporating user feedback earlier in the design process. Prototyping and iterative testing will help to refine the design and make sure it is aligned with user expectations.
